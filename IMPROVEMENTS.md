## 改善案（優先順）

### 1. 外部リンクの自動検出・スタイル化 ⭐⭐⭐

**現状の問題**
- 外部 URL は単なるテキストとして表示
- 他のリンクとの区別が不明確

**改善内容**
- 正規表現で http:// / https:// を自動検出
- 外部リンクに固有のアイコン表示（例：↗）
- target="_blank" rel="noopener" で新しいタブで開く
- CSS で内部リンクと区別してスタイル化

**実装例**

```python
# parser.py の _convert_markdown 後に追加メソッド
def _postprocess_links(self, html: str) -> str:
    """外部リンクを自動検出してスタイル適用"""
    pattern = r'((https?://)[^\s<>"')\]]+)'
    
    def replace_external(m):
        url = m.group(0)
        return f'<a href="{url}" target="_blank" rel="noopener" class="external-link">{url}</a>'
    
    return re.sub(pattern, replace_external, html)
```

```css
/* template.py の CSS に追加 */
.external-link {
    color: var(--accent);
    text-decoration: none;
}
.external-link::after {
    content: " ↗";
    font-size: 0.8em;
    opacity: 0.6;
}
```

### 2. 画像埋め込みの強化 ⭐⭐

**現状の問題**
- Obsidian 形式 ![[image.png]] のみ対応
- 通常の Markdown 画像記法未対応

**改善内容**
- ![alt text](image.png) 形式もサポート
- 画像クリックで拡大表示（ライトボックス）
- lazy loading 対応（loading="lazy"）
- 画像アスペクト比の自動計算

---

### 3. 本文の目次自動生成 ⭐⭐

**現状の問題**
- 長いドキュメントで見出しの把握が困難
- 特定のセクションへの素早い移動ができない

**改善内容**
- 見出し（h1~h6）を自動検出
- サイドバーに階層化された目次表示
- スクロール追従ハイライト（scrollspy）
- 現在表示中の見出しを自動選択

---

### 4. 前方参照リンクの強化 ⭐

**現状の問題**
- 存在しないページのリンクも問題なく生成される
- 破綻リンクの検出ができない

**改善内容**
- 未登録の wikilink を検出
- 404 風のエラーメッセージ表示
- 候補ページ（類似名）の提案

---

### 5. 検索機能の強化 ⭐

**現状の問題**
- 単純な部分一致のみ
- 検索結果の強調表示がない

**改善内容**
- 大文字・小文字を区別しない検索
- 正規表現検索オプション
- 検索結果のテキストハイライト
- 検索ワードの表示・クリア機能

---

### 6. 出力ファイルの最適化 ⭐

**現状の問題**
- CSS/JS が未圧縮でファイルサイズが大きい
- 埋め込み画像の重複データ

**改善内容**
- CSS/JS の圧縮（minification）
- 画像のベース 64 エンコード（外部依存なし）
- メタデータの追加（SEO 対策）
- キャッシュ制御ヘッダー
